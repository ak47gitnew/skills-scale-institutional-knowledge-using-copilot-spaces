# OctoAcme Project Management Docs

A central README that summarizes OctoAcme's project management approach and links to the detailed process documents in this repository.

Summary

OctoAcme follows an iterative, customer-first project management approach with clear ownership and lightweight artifacts. Key elements include:

- Initiation: short one-pager to capture problem, goals, and stakeholders
- Planning: prioritized backlog, estimates, dependencies, and a Definition of Done
- Execution: small PRs, CI checks, regular demos and standups
- Release: pre-release checks, automated deployments, and rollback plans
- Improvement: retrospectives, action items, and tracked improvements
