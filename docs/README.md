# OctoAcme Project Management Docs

This folder contains OctoAcme's project management process documents and a concise orientation for new teammates. The docs capture how we initiate and plan work, run execution, manage risks and communication, and improve continuously through retrospectives.

OctoAcme follows an iterative, customer-first approach. Work starts with a short Project One-pager to capture the problem, goals, success metrics, and stakeholders. Approved initiatives move into planning where the team breaks work into shippable backlog items with acceptance criteria, estimates, a Definition of Done, and a release/milestone plan. Planning artifacts and the backlog are used to guide sprint/iteration work and release scheduling.

Execution uses a project board workflow (Backlog 12 Ready 12 In Progress 12 In Review 12 QA 12 Done) and a disciplined pull request process that encourages small, testable changes, clear acceptance criteria, and automated CI checks before requesting review. Team rhythms include daily standups for blockers, weekly delivery syncs for progress and risk review, and end-of-sprint demos to validate outcomes. Risk management and communication are standardized with a simple risk register, weekly status templates, and clear escalation paths.

Quality assurance and release discipline are emphasized throughout: unit and integration tests, smoke tests for critical flows, security scans in CI, and manual QA as needed before release. Releases follow a checklist-driven process that includes staging verification, automated pipelines where possible, post-deploy checks, and documented rollback plans. Retrospectives capture learning and convert action items into issues for tracked follow-up.
