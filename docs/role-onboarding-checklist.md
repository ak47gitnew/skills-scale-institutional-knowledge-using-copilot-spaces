# Role Onboarding Checklist (Project-specific)

Use this to onboard a role (e.g., SRE, Data Analyst, UX) to a specific project.

- [ ] Role owner identified and added to project board / team.
- [ ] Role responsibilities and expectations documented in project README.
- [ ] Key contacts and reporting lines confirmed.
- [ ] Access granted to necessary tools, dashboards, and repos.
- [ ] Relevant process docs linked (release checklist, risk register, DoD).
- [ ] Initial sync scheduled with Product Manager and Project Manager.
- [ ] Deliverables and acceptance criteria for the role during the next milestone defined.
- [ ] Escalation path and on-call contacts documented.
